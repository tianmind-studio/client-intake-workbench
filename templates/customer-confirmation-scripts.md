# Customer Confirmation Scripts

Use these as drafts. Do not send them automatically.

## Scope Confirmation

```text
我先按目前材料理解一下范围：你需要我处理的是【项目类型/目标】，已有材料包括【源码/截图/数据库/要求】，最终要交付【交付物】。现在还缺【缺失信息】。你确认一下这个理解是否准确，我确认后再给你报价和时间。
```

```text
Here is my current understanding: the project goal is [goal], existing materials include [materials], and the expected deliverables are [deliverables]. I still need [missing information]. Please confirm whether this scope is accurate before I quote the timeline and price.
```

## Missing Information

```text
我还需要你补三样东西：1. 最终验收标准；2. 现有源码/文件包；3. 是否需要部署到服务器。补齐后我再判断工作量，避免先答应后面范围对不上。
```

```text
I still need three things: 1. final acceptance criteria; 2. the current source/files; 3. whether deployment is required. After that I can estimate the work more accurately.
```

## Quote Boundary

```text
这个报价按当前确认的范围来算，包含【包含内容】。如果后面新增功能、换方向或增加额外平台，需要重新确认时间和费用。
```

```text
This quote covers the confirmed scope: [included items]. New features, direction changes, or extra platforms should be estimated separately.
```

## Delivery Confirmation

```text
我这边已交付【交付内容】，包含【文件/链接/说明】。你先按确认范围检查一下；如果是范围内的问题我继续处理，如果是新增需求我们再单独确认。
```

```text
I have delivered [deliverables], including [files/links/notes]. Please check against the confirmed scope. I will fix in-scope issues; new requests can be estimated separately.
```
